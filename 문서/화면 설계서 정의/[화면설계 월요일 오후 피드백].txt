1. 관리자 메뉴 -> 그냥 조회 말고 더 구체적이고 직관적이게 ?? 무슨조회
2. 관리자 -> 삭제 이력 페이지 -> 관리자는 테이블을 사용함 (다시 페이지 만들기) 
3.  메인 게시판 파일 타입 명시
게시판에 업로드되는 파일의 유형을 더 상세하게 명시하도록 변경
예: JPG, PNG, PDF, DOCX 등 구체적 확장자 및 타입 표기
4. 수정 페이지 첨부파일 관리
수정 및 작성 페이지에서 첨부파일의 최대 첨부 개수를 명확히 표기 및 제한
이미 등록된 게시글의 첨부파일이 정확히 관리되도록 기능 보완
5. DB files 엔티티 개선
기존 파일 URL 저장 외에, 파일 타입 컬럼 추가하여 파일 관리 체계 강화
6. posts 엔티티에 대표 img 넣어주기 
7. 결과서 제출 프로세스
결과서를 제출할 때는 계획서를 빠르게 작성하여 결과서에 포함하도록 업무 프로세스 간소화
8. 테스트 방안 선택 기준
서비스 로직에 대한 단위 테스트는 JUnit 등의 유닛 테스트 도구로 수행할 것
러프하게 API 동작 확인 시에는 API를 구현 후 Postman 등으로 테스트 진행
업무 상황에 따라 JUnit과 Postman 중 적절한 방법을 선택하되,

JUnit : 개발자 중심, 자동화 및 정밀 검증에 적합
Postman : 빠른 API 검증 및 협업에 용이함
